# NPC主数据设计文档

> 版本：v0.1.15  
> 文档类型：设定 / 主数据 / 系统接口文档  
> 适用范围：第一章《旁支入道》、第二章《出走与立身》前置设计  
> 设定优先级：低于 `docs/canon/00_content_master.md`，高于旧版零散 NPC 草稿  
> 成人情缘规则：所有可触发亲密关系的角色均为 18 岁以上；亲密关系必须自愿；亲密过程镜头外处理；具体机制参考 `docs/design/romance_relationship_system.md`。  
> 本文件目的：统一主要 NPC 身份、唯一 ID、关系变量、生命周期、系统接口、阵营反应、Fallback 备用接口、盗术暴露规则、魔道污染规则与旧设定废弃声明。

---

## 一、统一原则

第一章的核心不是“谁是绝对好人、谁是绝对反派”，而是**旁支玩家在资源稀缺、身份压制、人情交易与灰色诱惑中选择自己的第一条修仙道路**。

本文件遵循以下原则：

| 原则 | 说明 |
|---|---|
| 一个角色一个核心功能位 | 避免同一个 NPC 同时承担家主、魔修、丹师、盗术导师等互相冲突身份 |
| NPC 是系统入口 | 每个主要 NPC 都绑定一个或多个玩法系统 |
| NPC 是路线诱因 | 玩家靠近不同 NPC，会自然进入不同路线 |
| NPC 有收益也有代价 | 不能只有奖励，没有风险和关系后果 |
| NPC 可被攻略、可被背叛、可失去 | 高关系必须伴随后续责任 |
| 正道、灰道、魔道清楚分层 | 盗术是灰道，不等于魔道；炼魂血祭才是禁忌魔道 |
| 成人情缘独立成系统 | 情缘线服务人物、系统与剧情，不写露骨过程 |
| 第一章主要 NPC 不死亡 | 第一章内主要 NPC 可失踪、被禁足、敌对、转移地点，但不因玩家选择直接死亡，避免剧情死锁 |
| 核心 NPC 决裂不等于系统永久关闭 | 若核心导师决裂，应开启高成本、低效率、高风险的 Fallback 备用接口 |

## 二、NPC唯一 ID 规范

### 2.1 命名规则

NPC 唯一 ID 使用以下格式：

```text
NPC_姓氏或势力_名字_序号
```

示例：

```text
NPC_SHEN_RUOLAN_001
NPC_SHEN_SHUANG_001
NPC_WEN_BUYU_001
```

### 2.2 ID 使用规则

| 规则       | 说明                             |
| -------- | ------------------------------ |
| 永不复用     | 被删除、重命名、废弃的 NPC_ID 不得再次使用      |
| 不随显示名改变  | NPC 显示名可改，NPC_ID 不改            |
| 存档引用 ID  | 所有关系变量、事件标记、NPC 状态必须使用唯一 ID 存储 |
| 新增只加新 ID | 新版本只能新增 ID，不应修改已有 ID           |
| 删除只标记废弃  | 删除 NPC 或事件时，应在“废弃 ID 归档”中记录    |
| 版本更新需迁移  | 涉及存档字段变化时，后续代码版本必须提供迁移逻辑       |

## 三、生命周期状态 Lifecycle State

NPC 不会永远处于同一状态。为防止剧情和系统死锁，所有主要 NPC 应具备生命周期状态。

| 状态         | 含义       | 系统处理                                      |
| ---------- | -------- | ----------------------------------------- |
| ACTIVE     | 正常可交互    | NPC 在对应地点出现，可触发事件                         |
| LOCKED     | 暂时不可交互   | 如闭关、禁足、受伤、被罚面壁，关系数据保留                     |
| EXILED     | 被逐出 / 出走 | 物理位置转移，交互入口改变                             |
| MISSING    | 失踪       | 暂不可交互，可触发调查事件                             |
| DEAD       | 死亡       | 第一章主要 NPC 禁用该状态；第二章后若使用，必须自动开启相关 Fallback |
| DEPRECATED | 设定废弃     | 仅用于旧设定归档，不进入新内容                           |

第一章规则：

```text
第一章所有主要 NPC 不进入 DEAD 状态。
玩家选择可导致 NPC 敌对、禁足、失踪、被审查、出走，但不能造成第一章主线死锁。
```

## 四、关系变量类型

不同 NPC 不应共用同一套“好感度”。应按功能类型区分变量。

| 类型  | 适用 NPC         | 核心变量                  |
| --- | -------------- | --------------------- |
| 情缘型 | 沈若兰、沈霜、沈青雀、柳听弦 | 好感、信任、情欲、占有、声誉风险、情债因果 |
| 导师型 | 沈怀安、沈霜、沈若兰     | 信任、认可、传授权限            |
| 交易型 | 鬼手老陈、闻不语       | 商誉/信用、欠债、违约记录、保密等级    |
| 权力型 | 沈正衡、族老会        | 关注度、警惕度、利用价值、裁决倾向     |
| 竞争型 | 沈云庭、沈子岳        | 认可、竞争度、仇恨、报复倾向        |
| 魔道型 | 沈墨阳、魔修残脉       | 魔染度、业力、诱惑度、追查风险       |

### 4.1 关系变量互斥规则

| 规则      | 说明                                |
| ------- | --------------------------------- |
| 信任与仇恨互斥 | 同一 NPC 的信任上升时，仇恨通常下降；仇恨大幅上升时，信任下降 |
| 情缘与敌对互斥 | 同一 NPC 不能同时处于“情缘绑定”和“敌对决裂”状态      |
| 信用与违约互斥 | 交易型 NPC 的违约记录会直接压低信用/商誉           |
| 关系有上下限  | 所有关系变量都应有上下限，不允许无限溢出              |
| NPC 非全知 | NPC 只能对自己知道、看到、被告知或被证据证明的事件作出反应   |

## 7.1 沈若兰

**NPC_ID**: `NPC_SHEN_RUOLAN_001`
**Lifecycle**: `ACTIVE`
**Relation_Type**: `ROMANCE`, `MENTOR`
**Core_System**: `System_Farming`, `System_Herb_Garden`, `System_Romance_Spirit_Breath`
**Fallback_ID**: `FB_FARMING_OLD_FARMER_001`
**Faction**: `Faction_Shen_Direct`, `Faction_Herb_Garden`
**Romance**: `YES`

### 基本定位

沈若兰是直系出身的药田区代理主管，主掌第一章灵田、灵植、药田改革与温和情缘线。她是最早认真看见玩家价值的直系角色。

### 身份支撑逻辑

沈若兰的母亲曾是药田一脉前任主管，留下过灵田轮作法、灵草病害手札和几份独门药田管理经验。沈家近年灵田减产严重，族老会虽不愿让年轻一代掌握太多资源，但若兰是少数真正懂灵植的人，因此被安排为“药田区代理主管”。

她有职位，但并非完全实权。她能决定低阶种子分配、巡田、病害处理和小范围试种，却不能改变旁支灵田租赁制度，也不能绕过族老会调动核心药田资源。

### 小传

沈若兰出身直系，却没有完全站在直系立场上。她自幼学习灵植、药理与家族灵田管理，对药田中旁支子弟的辛苦看得很清楚。

她不是反叛者，也不是天真善人。她知道家族秩序不会因为几句话改变，也知道直系资源倾斜背后有现实压力。但她无法接受旁支被当成一次性劳力消耗。

云渊历 1045 年，她被安排主导灵田区。此后，她逐渐发现灵田减产不只是天灾，而与家族分配、土壤透支、黑水坊市暗中吸走优质灵草有关。她曾向族老会提议降低旁支灵田租赁费，但被否决。

玩家若常去灵田区、照料灵田或帮助旁支，她会最早注意到玩家。随着互动加深，她会逐渐对玩家产生爱慕。这份感情很危险：她是直系，玩家是旁支；她的婚配可能被族老会利用，玩家也可能因为接近她而被直系盯上。因此，她越喜欢玩家，越会克制。

### 性格

| 层面 | 描述                      |
| -- | ----------------------- |
| 表面 | 温和、克制、礼貌，不轻易发怒          |
| 内在 | 有主见，长期压抑不满，不愿被家族安排成工具   |
| 感情 | 慢热但深，一旦动心很难放下           |
| 欲望 | 克制型，不轻浮，但高压下会产生强烈靠近冲动   |
| 优点 | 细心、讲理、愿意给旁支机会           |
| 缺点 | 顾虑太多，前期不敢公开对抗族老会        |
| 底线 | 不允许玩家伤害药田、出卖旁支、滥用魔道污染灵植 |

### 关键事件

| 事件ID                                        | 事件名    | 时间窗口    | 条件            | 成功后果        | 失败/背叛后果    |
| ------------------------------------------- | ------ | ------- | ------------- | ----------- | ---------- |
| EVT_SHEN_RUOLAN_001_FIELD_NOTICE            | 巡田留意   | 第2月起    | 玩家照料灵田        | 若兰注意玩家      | 无          |
| EVT_SHEN_RUOLAN_002_GRASS_TALK              | 草木相谈   | 第3月起    | 若兰好感提升，玩家灵植入门 | 灵植教学        | 判断错误则轻微降好感 |
| EVT_SHEN_RUOLAN_003_MOON_FIELD              | 月下心动   | 第5月起    | 玩家帮助药田或旁支     | 开启私人对话      | 讽刺她会降好感    |
| EVT_SHEN_RUOLAN_004_SPIRIT_GRASS            | 灵草托付   | 第6-9月   | 信任达标          | 变异灵草线开启     | 出卖灵草导致关系重创 |
| EVT_SHEN_RUOLAN_005_NIGHT_BEFORE_TOURNAMENT | 大比前夜巡田 | 第11-12月 | 高好感高信任        | 情缘确认，灵息双修入口 | 利用她则药田线断裂  |

### 可提供帮助

* 灵植熟练度提升
* 灵田产量提升
* 优质种子、灵草、药浴材料
* 降低虫害、枯萎、灵田异常风险
* 大比前提供心境恢复和辅助药材
* 高情缘后开启灵息双修

### 风险与代价

* 与她关系过近会增加直系关注
* 若关系暴露，沈子岳可能挑衅，族老会可能干预
* 亲密后背叛，药田线严重受损
* 若玩家污染灵田或出卖旁支，她会彻底心冷

### 第二章延伸

若关系良好，沈若兰可成为家族改革线核心人物。她可以选择留在沈家推动药田改革，也可能在极端情况下与玩家出走。若玩家背叛她，第二章她可能成为保守派利用的药田代表，与玩家保持冷淡甚至敌对。

## 7.2 沈云庭

**NPC_ID**: `NPC_SHEN_YUNTING_001`
**Lifecycle**: `ACTIVE`
**Relation_Type**: `RIVAL`, `POWER_EDGE`
**Core_System**: `System_Combat`, `System_Family_Rank`, `System_Tournament`
**Fallback_ID**: `FB_COMBAT_OLD_MANUAL_001`
**Faction**: `Faction_Shen_Direct`, `Faction_Martial_Lineage`
**Romance**: `NO`

### 基本定位

沈云庭是直系年轻精英，演武一脉代表。他是第一章玩家必须面对的资源优势者，代表“强者优先获得资源”的直系逻辑。

### 小传

云渊历 1044 年，沈云庭突破炼气七层，被家族视为下一代筑基种子。他接受了最好的族学训练、演武资源和功法传承，也因此天然认为“资源应该给最有可能成功的人”。

他不喜欢旁支，但这种不喜欢不是沈子岳式的粗暴羞辱，而是一种更冷静的轻视。玩家若表现平庸，他不会记住玩家；若表现突出，他会先警惕，再评估，最后可能认可。

### 性格

| 层面 | 描述                  |
| -- | ------------------- |
| 表面 | 冷静、自律、礼数周全          |
| 内在 | 家族责任感与直系优越感并存       |
| 优点 | 不滥用暴力，尊重真正实力        |
| 缺点 | 资源逻辑过强，容易把弱者视为低价值   |
| 底线 | 不能容忍玩家公开羞辱直系或破坏家族根基 |

### 关键事件

| 事件ID                                     | 事件名  | 时间窗口   | 条件         | 后果         |
| ---------------------------------------- | ---- | ------ | ---------- | ---------- |
| EVT_SHEN_YUNTING_001_MINOR_TEST          | 小考压场 | 第4月    | 家族小考       | 展示直系实力     |
| EVT_SHEN_YUNTING_002_ARENA_EDGE          | 演武场外 | 第4-7月  | 玩家围观/偷学/请教 | 获取战斗情报或被警惕 |
| EVT_SHEN_YUNTING_003_DIRECT_INVITE       | 直系邀请 | 第9-11月 | 玩家表现突出     | 进入直系观察线    |
| EVT_SHEN_YUNTING_004_TOURNAMENT_STANDOFF | 大比对峙 | 第12月   | 大比阶段       | 竞争或认可      |

### 第二章延伸

沈云庭可成为玩家的竞争者、盟友或家族线对手。若天罗宗施压，他可能代表直系主张强硬扩张；若玩家推动旁支改革，他会成为最难说服但最有价值的直系支持者。

## 7.3 沈子岳

**NPC_ID**: `NPC_SHEN_ZIYUE_001`
**Lifecycle**: `ACTIVE`
**Relation_Type**: `RIVAL`, `HOSTILE`
**Core_System**: `System_Combat`, `System_Tournament`, `System_Pressure`
**Fallback_ID**: `FB_RIVAL_GENERIC_001`
**Faction**: `Faction_Shen_Direct`, `Faction_Martial_Lineage`
**Romance**: `NO`

### 基本定位

沈子岳是直系子弟、演武一脉激进派，是第一章让玩家直接感受到直系压迫的人。

### 小传

沈子岳不如沈云庭沉稳，也不如沈若兰受人喜欢。他从小被灌输“直系天然高于旁支”，又长期活在沈云庭和沈若兰的光环下，因此对旁支格外蛮横。

他的母亲出身较低，虽被纳入直系体系，却常被旧派直系看不起。因此沈子岳比其他直系更急于证明自己“配得上直系身份”。他欺负旁支，一半是优越感，一半是焦虑。

### 被打服规则

| 胜利方式      | 沈子岳反应               |
| --------- | ------------------- |
| 堂堂正正击败    | 愤怒但开始认可             |
| 连续两次正面胜过他 | 开启“嘴硬认可”            |
| 救过他一次再击败他 | 可转为竞争型友方            |
| 用盗术暗算取胜   | 若未暴露短期获胜；若暴露，仇恨大幅增加 |
| 用毒丹/魔功取胜  | 视为卑劣，永久敌对           |
| 公开羞辱他     | 即使获胜，也埋下报复因果        |

### 压迫机制落地

| 行为    | 说明                 |
| ----- | ------------------ |
| 灵石勒索  | 第2-4月可能向玩家索要灵石     |
| 演武场挑衅 | 第7月公开挑衅旁支          |
| 资源挤压  | 可通过直系关系压低旁支练武场使用机会 |
| 败后报复  | 若被羞辱，可能散播流言或找人堵路   |

## 7.4 沈怀安

**NPC_ID**: `NPC_SHEN_HUAIAN_001`
**Lifecycle**: `ACTIVE`
**Relation_Type**: `MENTOR`
**Core_System**: `System_Foundation`, `System_Accumulation_Burst`, `System_Herb_Gathering`
**Fallback_ID**: `FB_FOUNDATION_SELF_CULTIVATION_001`
**Faction**: `Faction_Shen_Branch`, `Faction_Herb_Garden_Old_Line`
**Romance**: `NO`

### 基本定位

## 八、信息暴露度规则

玩家的行为不应被 NPC 全知。偷窃、黑水交易、魔道接触等灰色事件必须区分信息暴露度。

| 暴露度           | 含义              | 例子            | 后果          |
| ------------- | --------------- | ------------- | ----------- |
| Secret 隐秘     | 无人发现，无痕迹，无追踪标记  | 完美盗取直系灵石      | 无公开后果       |
| Traceable 可追查 | 有人知道东西丢了，但不知道是谁 | 丹房少了一瓶药       | 区域警戒提高      |
| Public 公开     | 多人目击或公开场合发生     | 大比中使用魔功       | 立即触发阵营反应    |
| Leaked 泄露     | 情报被第三方出售或传播     | 闻不语卖出玩家黑水交易记录 | 从隐秘转为可追查或公开 |
| Evidence 证据确凿 | 赃物、痕迹、人证俱在      | 赃物被搜出         | 直接结算惩罚      |

核心规则：

```text
未被发现 = 不触发公开声望惩罚、NPC好感惩罚、族老会处罚。
未暴露时，只可能增加痕迹、追踪标记、隐性恶因、物品污染、心理压力等隐藏变量。
公开惩罚只在被发现、被出卖、证据暴露、追踪成功时结算。
沈青雀不是全知 NPC，她只有在亲眼发现、玩家承认、或事件泄露后才会反应。
天机阁也不是全知，除非玩家留下可交易情报、被人目击、或触发追踪标记。
```

| 获取方式 | 沈青雀教学、黑水残卷、鬼手老陈旧货、天机阁情报 |
| 风险   | 被高阶探查术识破后，惩罚更重          |
| 路线价值 | 盗术流、黑水流、魔道伪装流、随心流都可用    |

### 11.2 熟练度分层

| 熟练度 | 效果                                     |
| --- | -------------------------------------- |
| 入门  | 可隐藏少量小物件气息，降低被搜出概率                     |
| 熟练  | 可隐藏储物袋中部分赃物气息，同境界普通探查难以发现              |
| 精通  | 可压制自身修为波动，伪装为低一层境界                     |
| 大成  | 同境界几乎无法探查；可稳定伪装境界、隐藏大部分物品              |
| 圆满  | 高一大境界也难以看破；可短时间伪装气息来源，隐藏魔气或黑水追踪标记，但消耗大 |

### 11.3 探查对抗规则

| 玩家隐匿等级 | 普通同境界探查 | 高一小层探查 | 高一大境界探查 | 天机阁/特殊法宝    |
| ------ | ------- | ------ | ------- | ----------- |
| 入门     | 可能发现    | 容易发现   | 基本发现    | 必然发现        |
| 熟练     | 较难发现    | 可能发现   | 容易发现    | 高概率发现       |
| 精通     | 很难发现    | 较难发现   | 可能发现    | 可能发现        |
| 大成     | 基本无法发现  | 很难发现   | 有概率发现   | 仍可检定        |
| 圆满     | 无法发现    | 基本无法发现 | 很难发现    | 仍有极低概率或剧情检定 |

### 11.4 可隐藏对象

| 隐藏对象 | 入门  | 熟练   | 精通     | 大成      | 圆满       |
| ---- | --- | ---- | ------ | ------- | -------- |
| 少量灵石 | 可以  | 稳定   | 稳定     | 稳定      | 稳定       |
| 普通赃物 | 有风险 | 可以   | 稳定     | 稳定      | 稳定       |
| 丹药气息 | 不稳定 | 可以   | 稳定     | 稳定      | 稳定       |
| 功法残页 | 不稳定 | 可以   | 稳定     | 稳定      | 稳定       |
| 魔道残物 | 不可  | 高风险  | 可短时压制  | 可隐藏但有反噬 | 可稳定短时隐藏  |
| 自身修为 | 不可  | 轻微压制 | 可伪装低一层 | 可伪装多个层级 | 可伪装成指定区间 |
| 追踪标记 | 不可  | 延迟触发 | 可压制    | 可转移/伪装  | 可短时欺骗来源  |

### 11.5 使用代价

| 使用方式   | 代价            |
| ------ | ------------- |
| 隐藏普通物品 | 消耗灵气          |
| 隐藏赃物   | 消耗灵气 + 增加疲劳   |
| 隐藏丹药气息 | 消耗灵气 + 可能影响药性 |
| 隐藏魔气   | 消耗心境 + 增加污染风险 |
| 伪装修为   | 降低当月修炼效率      |
| 长时间维持  | 容易触发气息紊乱      |
| 被强行看破  | 直接触发更严重怀疑     |

## 十二、魔道接触分阶与清除代价

### 12.1 魔道接触阶段

| 阶段     | 行为        | 后果             |
| ------ | --------- | -------------- |
| 听闻传言   | 只听到魔道消息   | 无污染，只增加知识      |
| 接触魔道残物 | 碰到魔器、魔修遗物 | 轻微业力，可清除       |
| 阅读魔功残页 | 阅读禁忌文字    | 心魔风险增加，可压制     |
| 初次修炼魔功 | 主动运转魔功    | 根基出现污染标记，清除代价高 |
| 炼魂/血祭  | 使用禁忌手段    | 深度魔染，基本不可逆     |
| 公开使用魔功 | 大比或公开场合暴露 | 进入逃亡/魔道线       |

### 12.2 清除代价

| 阶段     | 清除方法          | 代价           |
| ------ | ------------- | ------------ |
| 听闻传言   | 无需清除          | 无            |
| 接触魔道残物 | 沈怀安清心法、正道任务   | 轻微，消耗时间      |
| 阅读魔功残页 | 沈怀安 + 沈若兰联手压制 | 中等，心境下降      |
| 初次修炼魔功 | 特定奇遇 + 大量功德   | 高，永久根基微损     |
| 炼魂/血祭  | 几乎不可逆         | 极高，可能永久进入魔道线 |
| 公开使用魔功 | 不可逆，逃亡        | 进入魔道/逃亡结局    |

### 12.3 魔道污染阶段表现

| 阶段    | 表现                            |
| ----- | ----------------------------- |
| 无明显表现 | 只有沈怀安、沈正衡等极少数人可能察觉            |
| 轻微污染  | 修炼速度小幅提升，心魔事件概率增加             |
| 中度污染  | 修炼速度提高，心魔事件明显增多，部分正道 NPC 态度冷淡 |
| 重度污染  | 修炼速度大幅提高，心魔事件频繁，正道 NPC 高度警惕   |
| 深度魔染  | 无法进入沈家核心区域，大比时可能被沈正衡判定为魔道     |

设计说明：

```text
魔道不是一碰就永久锁死正道，而是越走越难回头。
轻度接触可补救，深度炼魂/血祭基本不可逆。
```

## 十三、Fallback备用接口

### 13.1 设计原则

```text
核心 NPC 决裂 ≠ 系统永久关闭。
但系统会转入高成本、低效率、高风险的备用接口。
Fallback 接口不会提供核心 NPC 的专属剧情、专属奖励或高阶传承。
```

### 13.2 触发规则

| 规则    | 说明                                       |
| ----- | ---------------------------------------- |
| 决裂触发  | 核心 NPC 好感/信任≤0 且触发决裂事件后，自动解锁对应 Fallback  |
| 可修复关系 | 部分决裂可通过特殊任务修复，修复后主接口恢复，Fallback 可保留但效率较低 |
| 极端决裂  | 如出卖沈青雀导致她被抓，可能永久关闭原接口                    |
| 成本更高  | Fallback 成本永远高于正常接口                      |
| 无专属内容 | Fallback 不提供专属剧情、情缘、独门技巧                 |

### 13.3 炼丹系统 Fallback

| 接口层级 | NPC/渠道     | 条件      | 效率   | 风险           |
| ---- | ---------- | ------- | ---- | ------------ |
| 主接口  | 沈霜         | 信任达标    | 100% | 低            |
| 备用A  | 黑水残缺丹方     | 灵石、黑水商誉 | 60%  | 炸炉率上升、丹毒风险   |
| 备用B  | 旁支老丹师（待实装） | 旁支声望、灵石 | 50%  | 只能炼基础丹药      |
| 禁用态  | 购买成品丹      | 无导师、无渠道 | 低    | 成本高、无法提升炼丹熟练 |

流程：

```text
玩家失去沈霜 → 炼丹系统不关闭 → 转入黑水残缺丹方或旁支老丹师 → 成功率下降、风险增加 → 玩家仍可继续但代价更高
```
