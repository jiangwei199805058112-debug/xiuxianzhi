## 一、文档定位

`10_chapter1_event_pool.md` 用于存放第一章可重复调用、可随机触发、可条件触发的小型事件。

它和路线事件链文档的区别如下：

| 文档 | 作用 |
|---|---|
| `07_chapter1_main_arc.md` | 第一章整体月度主线、章节安全网、信息暴露度、大比与结局入口 |
| `09_chapter1_route_event_chains.md` | 九条主要玩法路线的 12 个月主干推进 |
| `10_chapter1_event_pool.md` | 可被系统抽取的小事件、NPC反应事件、补救事件、危机事件 |
| `11_tournament_event_design.md` | 第12月家族大比、对手、判定、看台反应与结局衔接 |

本文件的核心目标：

1. 让第一章每个月不只有固定主线，也有可变化的小事件。
2. 让 NPC 不只在主线出现，也能根据玩家行为做出月度反应。
3. 让盗术痕迹、黑水债务、魔道污染、丹毒、灵田虫害等风险有明确事件承接。
4. 让玩家误操作后有补救机会，不因一次错误死档。
5. 为后续事件 JSON/YAML 配置化提供统一结构。

后续所有事件建议使用统一字段，方便转为 JSON/YAML 配置。

| 是否强制 | 否 |

### 剧情文本

玩家在田埂边发现几片被咬碎的清心叶，叶背还有细小虫卵。

### 选项

| 选项 | 结果 |

## 三、事件池分类

第一章事件池分为以下类型：

| 类型 | 说明 |
|---|---|
| 月度随机事件 | 每月可能触发的小事件 |
| NPC好感事件 | 根据 NPC 好感/信任触发 |
| 路线专属事件 | 某条路线投入达到条件后触发 |
| 危机事件 | 痕迹、污染、债务、丹毒等过高时触发 |
| 补救事件 | 玩家犯错后提供修复机会 |
| 大比准备事件 | 第10-11月集中触发 |
| 大比看台反应 | 第12月根据路线和 NPC 关系触发 |
| 随心混合事件 | 多系统浅尝、误操作、杂学组合事件 |

## 四、通用触发规则

### 4.1 事件触发优先级

同一月事件触发建议按以下顺序：

| 优先级 | 事件类型 | 说明 |
|---|---|---|
| P0 | 强制主线事件 | 第一章月度主线必须优先 |
| P1 | 危机事件 | 高风险变量超过阈值时优先触发 |
| P2 | NPC主动反应事件 | NPC根据玩家行为反馈 |
| P3 | 路线专属事件 | 与玩家当前路线相关 |
| P4 | 随机小事件 | 填充月度变化 |
| P5 | 氛围事件 | 只提供文本和少量反馈 |

### 4.2 单月事件数量建议

| 事件类型 | 每月建议数量 |
|---|---:|
| 强制主线 | 0-1 |
| 路线推进事件 | 0-1 |
| NPC反应事件 | 0-2 |
| 随机事件 | 0-2 |
| 危机/补救事件 | 根据变量触发 |

一个月内不应堆叠太多强事件。若事件过多，应优先保留主线、危机、路线事件，延后氛围事件。

### 4.3 重复事件规则

| 规则 | 说明 |
|---|---|
| 可重复事件 | 虫害、价格波动、小额失误等 |
| 不可重复事件 | 首次见面、第一次误操作、大比前确认等 |
| 冷却事件 | 触发后隔1-3月才可再次触发 |
| 升级事件 | 同类事件多次触发后升级为危机事件 |
