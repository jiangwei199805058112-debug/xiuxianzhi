# 系统总规划

## 设计原则

第一章系统应服务“旁支入道”主题。每个系统都要回答三个问题：
- 它给玩家什么选择？
- 它留下什么代价？
- 它如何在大比或后续章节中被承认？

## 当前系统组

已存在系统：
- 基础属性与境界。
- 行动与月末事件。
- 灵田、炼丹炉、轻量装备。
- 普通坊市与黑水坊市。
- 盗术。
- 根基、熟练度、博学度、融会、厚积薄发。
- NPC 好感。
- 家族大比。

## 长期系统层级

第一层：行动收益。玩家每月通过行动获得资源、属性、熟练度和风险。

第二层：路线识别。系统通过字段判断玩家偏向炼丹、盗术、魔道、黑水、厚积薄发等路线。

第三层：剧情反馈。NPC、月末事件和大比评价根据路线与风险给出文本和结果。

第四层：章节承接。第一章结局标记进入第二章，影响入口和初始关系。

## 配置化方向

后续可逐步配置化：
- NPC 阶段事件。
- 地点事件池。
- 功法与残页。
- 黑水商品与服务。
- 灵田作物。
- 炼丹炉与丹方。
- 路线评价文案。

## 不宜过早复杂化

- 不做复杂装备强化。
- 不做大型宗门管理。
- 不做全地图开放。
- 不把盗术高阶做成稳定按钮。
- 不让灵田成为灵石矿。

## 第一章核心闭环

行动选择 -> 资源与熟练度 -> 风险与人情 -> 大比评分 -> 结局标记 -> 后续章节入口。
