# 成长系统规划

## 目标

让玩家长期使用某类行动时形成“会做这件事”的反馈，而不是只有一次性数值收益。

## 核心字段

- 根基：底层修行质量。
- 熟练度：各行动门类经验。
- 博学度：多门熟练度的动态综合。
- 融会状态：多门条件达成后的自动解锁。
- 突破感悟：突破时的方向选择。
- 厚积薄发：大比前的一次性综合爆发。

## 行动映射

打坐：根基、修炼熟练度。

采药：根基、采药熟练度。

炼丹：炼丹熟练度。

灵田：根基、灵植熟练度。

坊市：坊市熟练度，部分情报熟练度。

情报：情报熟练度。

结交族人：人情熟练度。

盗术：盗术熟练度。

魔道：魔道熟练度，少量斗法熟练度。

法术训练：斗法熟练度。

## 融会状态设计

融会状态不应大量直接加属性。更合适的效果：
- 小幅提高对应行动稳定性。
- 降低小概率负面。
- 大比对应关卡小幅加成。
- 解锁特殊文本或事件选择。

## 厚积薄发定位

厚积薄发应该是慢热综合路线的稀缺事件。它需要明确门槛，并且不能替代常规成长。

触发后：
- 给大比三关小幅综合加成。
- 高业力、高魔气、高暴露、高追踪会打折。
- 不清除负面标记。

## 风险

若博学度加成过高，所有路线都会变强。

若融会状态加分过宽，厚积薄发会失去稀缺性。

若条件过苛刻，玩家会感觉长期经营无回报。

## 后续改进

- 让 UI 更清楚显示距离融会条件还差什么。
- 为每个融会状态增加 1 到 2 个专属事件。
- 将融会条件和效果逐步配置化。
